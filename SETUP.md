# Crucible · Setup Walkthrough

End-to-end: from a fresh laptop to a working app icon on your phone home screen. Plan ~45 minutes the first time. Total cost: $0.

## What you'll need

- A computer with a terminal
- Your GitHub account (you said you have one)
- 15 minutes of focus

## 1 · Local prerequisites

You need **Node.js 20+** and **pnpm** (or npm — pnpm is faster).

```bash
# Check if you have Node already
node -v   # should print v20.x or higher

# If not, install via the official installer:
#   https://nodejs.org/en/download
# or via Homebrew on macOS:
brew install node@20

# Install pnpm:
npm install -g pnpm
```

## 2 · Create a Supabase project (free)

1. Go to https://supabase.com → **Start your project** → sign in with GitHub.
2. Click **New project**.
   - Name: `crucible`
   - Database password: generate one and **save it in your password manager** — you'll need it for `DATABASE_URL`.
   - Region: pick the one closest to Austin (`us-east-2` Ohio, or `us-west-1` California).
   - Plan: **Free**.
3. Wait ~2 minutes for the project to provision.
4. Once it's up, go to **Project Settings → API**. You'll need two values:
   - **Project URL** → this is your `NEXT_PUBLIC_SUPABASE_URL`
   - **Project API keys → `anon` `public`** → this is your `NEXT_PUBLIC_SUPABASE_ANON_KEY`
5. Go to **Project Settings → Database → Connection string → Transaction pooler** (port 6543). Copy that string and replace `[YOUR-PASSWORD]` with the password from step 2 — this is `DATABASE_URL`.

### Run the schema

In the Supabase dashboard, open **SQL Editor → New query**, paste the contents of `supabase/migrations/0000_init.sql`, and **Run**. You should see green ticks for each statement. This creates the `profiles`, `fasts`, `water_logs`, `weight_logs`, `fasting_templates`, `fasting_protocols` tables, sets up RLS, and seeds the protocol options.

### Configure auth

- **Authentication → Providers → Email** → make sure **Enable Email** is on. Toggle **Confirm email** off for the first run (faster to test). Magic-link sign-in is the default.
- **Authentication → URL Configuration** → set **Site URL** to `http://localhost:3000` for now. After you deploy, you'll add your Vercel URL too.

Now install deps and copy the env template:

```bash

```

Edit `.env.local` and paste in the three values from Supabase:

```

NEXT_PUBLIC_SUPABASE_ANON_KEY=eyJhbGciOi...
```

Boot the dev server:

```bash

```

Open http://localhost:3000. You'll be redirected to `/login`. Enter your email, hit **Send magic link**, check your inbox, click the link, and you'll land on the Today screen.

If anything errors, run `pnpm typecheck` to surface what's wrong.

## 5 · Deploy to Vercel (free)

1. Go to https://vercel.com → **Sign up with GitHub**.
2. Click **Add New… → Project** → import your `crucible` repo.
3. **Framework preset:** Next.js (auto-detected).
4. **Environment variables** — add the three from `.env.local`:
   - `NEXT_PUBLIC_SUPABASE_URL`
   - `NEXT_PUBLIC_SUPABASE_ANON_KEY`
   - `DATABASE_URL`
5. Click **Deploy**. Wait ~90 seconds.

You'll get a URL like `https://crucible-xyz.vercel.app`.

### Connect the URL back to Supabase

- **Supabase → Authentication → URL Configuration:**
  - **Site URL:** `https://crucible-xyz.vercel.app`
  - **Redirect URLs (allow list):** add both `https://crucible-xyz.vercel.app/auth/callback` and `http://localhost:3000/auth/callback`.

Without this, magic links from the deployed site will redirect you to localhost and fail.

## 6 · Install on your phone

On iPhone (iOS 16.4+ for full PWA features):

1. Open the Vercel URL in **Safari** (must be Safari for install).
2. Tap the **Share** icon (square with up-arrow) → scroll → **Add to Home Screen**.
3. Tap **Add**. Crucible is now an icon on your home screen — full-screen, no browser chrome, looks native.

Sign in with the magic link from the home-screen app.

On Android (Chrome): visit the URL → menu → **Install app**. Same result.

## 7 · Generate proper app icons (optional but recommended)

Right now there's a placeholder SVG in `public/icons/icon.svg`. iOS will use a default if it can't find PNGs. To get a clean home-screen icon:

```bash
# macOS:
brew install imagemagick
cd public/icons
magick icon.svg -resize 192x192 icon-192.png
magick icon.svg -resize 512x512 icon-512.png
magick icon.svg -resize 512x512 -background "#0b0f14" -gravity center -extent 512x512 icon-maskable-512.png
magick icon.svg -resize 180x180 apple-touch-icon.png
```

Or upload `icon.svg` to https://realfavicongenerator.net and download the package.

Commit, push, Vercel auto-redeploys, re-add to home screen.

## Troubleshooting

| Symptom | Fix |
|---|---|
| `Invalid API key` on sign-in | Triple-check `NEXT_PUBLIC_SUPABASE_ANON_KEY` — easy to truncate when copying |
| Magic link redirects to localhost from deployed site | Add the Vercel URL to **Supabase → Auth → URL Configuration → Redirect URLs** |
| `relation "profiles" does not exist` | Re-run `supabase/migrations/0000_init.sql` in the SQL editor |
| `prepared statement` errors | Confirm `DATABASE_URL` uses port `6543` (transaction pooler), not `5432` |
| Service worker stuck on old version | Bump `CACHE_VERSION` in `public/sw.js`, redeploy, hard-refresh |
| iOS install missing | Must be in Safari (not Chrome on iOS), and iOS 16.4+ for push notifications |
